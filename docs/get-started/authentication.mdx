# Authentication & providers

OpenAgent is **multi-provider**. You do not need a Google account to use it.
Choose local models (no key), free-tier cloud keys, or paid frontier keys — or mix
them and switch with `/models`.

---

## Quick decision table

| You want… | Do this | Key required? |
| --------- | ------- | ------------- |
| Maximum privacy / offline | [Local models](./local-models.md) (Ollama / LM Studio) | No |
| Zero or low cost | [Free models](./free-models.md) + `--free` | Free-tier keys (varies) |
| Best quality | BYOK OpenAI / Anthropic / Gemini / … | Yes |
| Fast iteration on free GPUs | Groq, Cerebras, OpenRouter `:free` | Usually yes |

---

## Interactive setup (recommended)

```bash
npm start -- --byok
# or
openagent --byok
```

The walkthrough:

1. Lists providers and whether a key is already present.
2. Prompts for missing keys.
3. Writes them to **`.env`** in the project (or user config, depending on mode).
4. Shows which models just became available.

In a live session:

```text
/byok                      # status for all providers
/byok groq gsk_...         # save one provider key
/models                    # picker (vision / streaming / key badges)
```

## Environment variables

Set these in a `.env` file at the repo root (see `.env.example`) or export them
in your shell.

| Provider | Environment variable | Notes |
| -------- | -------------------- | ----- |
| **Ollama** | — | Local default at `http://localhost:11434` |
| **LM Studio** | — | Local OpenAI-compatible server (often `:1234`) |
| **OpenAI** | `OPENAI_API_KEY` | Frontier models |
| **Anthropic** | `ANTHROPIC_API_KEY` | Claude family |
| **Gemini** | `GEMINI_API_KEY` | Google AI Studio key / free tier |
| **Groq** | `GROQ_API_KEY` | Very fast; free tier TPM limits apply |
| **DeepSeek** | `DEEPSEEK_API_KEY` | Low-cost frontier |
| **NVIDIA** | `NVIDIA_API_KEY` | OpenAI-compatible endpoint |
| **Together AI** | `TOGETHER_API_KEY` | Hosted open models |
| **Hugging Face** | `HF_TOKEN` | Rate-limited free inference |
| **OpenRouter** | `OPENROUTER_API_KEY` | Many `:free` and paid models |
| **Cerebras** | `CEREBRAS_API_KEY` | Fast free public endpoints (rate-limited) |
| **Z.ai** | `Z_AI_API_KEY` | GLM models |

## Optional: Gemini / Google sign-in

If you prefer Google account login or Vertex AI for Gemini specifically, you can
still use those flows when enabled in your build. For most OpenAgent users the
**API key + multi-provider** path above is simpler and covers every backend.

| Scenario | Approach |
| -------- | -------- |
| Gemini via AI Studio | Set `GEMINI_API_KEY` |
| Other providers | Matching `*_API_KEY` |
| Local only | No key — run Ollama |

## Selecting a model after auth

```bash
# By registry key (configs/models.toml)
npm start -- -m groq-llama-3.1-8b
npm start -- -m openrouter-free
npm start -- -m cerebras-gpt-oss-120b

# By provider
npm start -- --provider ollama
npm start -- --provider groq -m llama-3.1-8b-instant

# Free catalog rotation
npm start -- --free "your task"
```

In-session: `/models` or `/models set <name>`.

Full model list and vision matrix: [Models.MD](../../Models.MD) ·
[Provider catalog](./providers.md).

## Security tips

- Prefer **local models** for private codebases.
- Never commit `.env` or paste keys into issues/chat logs.
- Use [Trusted folders](../cli/trusted-folders.md) and
  [Sandboxing](../cli/sandbox.md) for untrusted workspaces.
- Avoid `--yolo` / auto-approve outside isolated directories.
