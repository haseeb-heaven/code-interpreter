# Installation

Install OpenAgent on macOS, Linux, or Windows and verify it starts cleanly.

---

## System requirements

| | Minimum | Comfortable |
| -- | -- | -- |
| **OS** | Windows 10+, macOS 12+, Ubuntu 20.04+ | Latest LTS / Windows 11 |
| **Runtime** | Node.js **22.0.0+** | Latest Node.js LTS |
| **RAM** | 4 GB (short sessions) | 16 GB (large repos / long context) |
| **Shell** | Bash, Zsh, or PowerShell | — |
| **Network** | Optional for **local** models | Required for cloud providers |

> [!NOTE]
> Fully offline use is supported when Ollama or LM Studio is running locally.
> Cloud providers need outbound HTTPS and the matching API key.

---

## Install OpenAgent

<Tabs>
  <TabItem label="From source">

  Best for contributors and the latest `feature` / `docs` branch work:

  ```bash
  git clone https://github.com/haseeb-heaven/open-agent.git
  cd open-agent
  npm install
  npm run build
  npm start
  ```

  Useful scripts:

  ```bash
  npm start -- --free "hello"
  npm start -- --models
  npm start -- --byok
  npm test
  ```

  </TabItem>
  <TabItem label="npm (global)">

  ```bash
  npm install -g open-agent
  openagent
  openagent --version
  ```

  Update later:

  ```bash
  npm update -g open-agent
  # or, if supported in your build:
  openagent update
  ```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

  ```powershell
  # Requires Node.js 22+ from https://nodejs.org
  node -v

  git clone https://github.com/haseeb-heaven/open-agent.git
  cd open-agent
  npm install
  npm run build
  npm start
  ```

  Global install:

  ```powershell
  npm install -g open-agent
  openagent
  ```

  </TabItem>
  <TabItem label="Anaconda / restricted env">

  ```bash
  conda create -y -n openagent_env -c conda-forge nodejs=22
  conda activate openagent_env
  npm install -g open-agent
  openagent
  ```

  </TabItem>
</Tabs>

## Verify the install

```bash
# Interactive TUI
openagent
# or from a clone:
npm start

# Headless smoke check
openagent -p "reply with the single word: pong"
# or:
npm start -- -p "reply with the single word: pong"
```

You should see the **OA** header and a prompt (interactive), or a short model
reply (headless). If the model call fails, you still verified the binary starts —
fix providers next via [Authentication](./authentication.mdx).

## Optional: local model backend

For zero-key usage:

1. Install [Ollama](https://ollama.com/).
2. Pull a model: `ollama pull llama3.1:8b`
3. Run OpenAgent — it prefers Ollama when no cloud key is configured.

See [Local models](./local-models.md).

---

## Environment file

Copy the example env and add only the keys you need:

```bash
cp .env.example .env
```

Or use the interactive walkthrough:

```bash
npm start -- --byok
# or
openagent --byok
```

Never commit `.env`. It is gitignored.
