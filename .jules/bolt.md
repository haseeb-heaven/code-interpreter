## 2024-05-16 - Pre-compiling Regexes in Safety Manager Loop
**Learning:** In highly trafficked areas of the codebase, like the `ExecutionSafetyManager.assess_execution` method which parses and evaluates execution boundaries on almost every operation, dynamically compiling list-based regular expressions (e.g. `any(re.search(p, ...) for p in list)`) introduces measurable overhead. Using `re.compile()` inside loops or list comprehensions causes redundant compilation.
**Action:** When working on performance improvements, specifically search for loops or frequent checks that evaluate raw regex strings. Move these to class-level attributes using generator expressions mapped to tuples (e.g., `_COMPILED_PATTERNS = tuple(re.compile(p) for p in _PATTERNS)`), and use the `.search()`, `.match()`, and `.findall()` methods directly on the compiled objects instead of module-level `re.*` functions.
