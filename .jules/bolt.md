## 2024-06-15 - Regex Compilation Optimization
**Learning:** In Python 3, list comprehensions inside a class body do not have access to the class's scope, leading to `NameError`. Furthermore, repeatedly compiling large numbers of complex regular expressions using `re.search()` instead of `re.compile()` in hot paths (like system safety checks) adds measurable overhead, even with Python's internal regex caching.
**Action:** When working with many regular expressions that are evaluated repeatedly (such as safety checks applied to every string of generated code), pre-compile them at the class level using a generator expression converted to a tuple, like `tuple(re.compile(p) for p in _PATTERNS)`, and use `.search()` directly on the compiled pattern object. This avoids scope errors and eliminates redundant compilation overhead.
