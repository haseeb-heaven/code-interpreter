## 2024-05-11 - Command Injection via subprocess.call\n**Vulnerability:** Command injection when opening resource files on Windows due to using `subprocess.call(['start', filename], shell=True)`.\n**Learning:** The `start` command is a shell built-in on Windows, which necessitates `shell=True`. When `shell=True` is used with unsanitized user-controlled variables (like `filename`), it allows command injection if the variable contains shell metacharacters (e.g., `&`).\n**Prevention:** Replace `subprocess.call(['start', filename], shell=True)` with Python's native `os.startfile(filename)`, which safely delegates file execution to the OS without invoking a shell layer.
